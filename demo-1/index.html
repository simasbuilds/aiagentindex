Demo #1: Project Analyzer Agent.

A live build showing how Claude can read a project's PDF documentation and become an expert on its scope, requirements, and workflows — cutting Project Manager onboarding from days to minutes.

Live demo · presentation content

Build the agent on stage.

Use this page as the participant-facing companion during the demonstration — the same inputs and prompt you'll use live, so the room can follow along or try it afterward.

01 · Scenario

Business problem

Faster onboarding through documentationNew Project Managers often spend days reading requirements, specs, and workflow docs before they can contribute. This demo builds a Project Analyzer Agent: give Claude a project's PDF documentation and it becomes an expert on scope, goals, requirements, and workflows — ready to brief a new team member in minutes.
02 · Inputs

Source materials

Project knowledge base — 3 PDFsDownload the documents and place them in an assets folder next to the agent's CLAUDE.md. They are the agent's complete and only knowledge base.
Open the PDF folder ↗
03 · Starting prompt

Paste this to build the agent

This is the exact prompt used to generate the agent's CLAUDE.md and skill.md. Copy it, drop it into Claude alongside the assets folder above, and it's ready for anyone in the room to try.

ROLEYou are a senior Project Management expert.
TASKBuild a Project Analyzer Agent that demonstrates how AI can understand an enterprise project by reading its documentation. The PDFs in the assets folder are the project's complete knowledge base — review them fully and become an expert on the project before answering anything.
CLAUDE.MDDefine Claude's role as a Senior Project Manager. All project knowledge comes from the assets folder. Build a complete understanding before answering. Always reference the source documents, never invent details, connect information across documents, and explain concepts clearly for both technical and non-technical stakeholders.
SKILL.MDDefine a Project Analyzer Agent that understands scope, business goals, functional and business requirements, processes, workflows, user stories, acceptance criteria, testing scenarios, terminology, and stakeholders — and can explain the project to new teammates, answer questions in natural language, summarize documentation, connect information across documents, run impact analysis, and generate executive summaries and stakeholder updates.
EXAMPLE PROMPTSSummarize this project. Explain the project in five minutes. What business problem is this solving? Who are the stakeholders? Explain the onboarding workflow. Show me all user stories related to identity verification. Which requirements are affected if identity verification changes? What testing scenarios exist? Generate an executive summary. Explain this project to a new Project Manager.
GOALA polished, enterprise-quality Project Analyzer Agent a Project Manager could realistically use when joining a new project.
04 · Live build

Presentation sequence

  1. Create a project folder with an assets subfolder and drop in the 3 PDFs.
  2. Open Claude in that folder and paste the starting prompt above.
  3. Let Claude draft CLAUDE.md and skill.md, then review them together.
  4. Ask a real project question live — e.g. "Explain the onboarding workflow."
  5. Show the agent answering with citations back to the source PDFs.
05 · Final output

Completed example

A working Project Analyzer AgentA CLAUDE.md defining the agent's role and ground rules, a skill.md defining its capabilities, and a live Q&A exchange where it explains the project and cites its sources — proof a new PM could get productive in minutes, not days.
06 · Debrief

Discussion questions

Ask the roomWhat did the agent get right without manual explanation? Where did it need a correction or an extra document? How much faster was this than a human-led walkthrough? What governance would you want before trusting agent answers on a real project?

Narrate decisions

Explain why the prompt, the assets folder, and each instruction exist — not only what you click.

Show one failure

A controlled gap in the documents teaches the room how to inspect and correct agent answers.

Close the loop

Connect the agent's answer back to the original onboarding problem and where a human still needs to review.
